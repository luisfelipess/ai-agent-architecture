# AI Agent Architecture: A Field Guide

A three-volume technical reference on integration patterns, execution architecture, and workspace design for AI-powered software.

**→ [Read the full series at luisfelipess.github.io/ai-agent-architecture](https://luisfelipess.github.io/ai-agent-architecture/)**

---

## What's inside

| Volume | Question | Topic |
|--------|----------|-------|
| [Vol 1](https://luisfelipess.github.io/ai-agent-architecture/vol1/00-overview/) | *Which integration pattern should I use?* | MCP, Local Tools, Skills, Steering Files, Hooks |
| [Vol 2](https://luisfelipess.github.io/ai-agent-architecture/vol2/00-overview/) | *How do I build the system to be accurate and self-improving?* | Progressive Context Loading, Modular Design, Accuracy Flywheel |
| [Vol 3](https://luisfelipess.github.io/ai-agent-architecture/vol3/00-overview/) | *How should the workspace the agent operates in be designed?* | Model A vs Model B, Policy in Code, Tool-Surfaced Contracts |

The series grew out of hands-on experience building AI-powered tools and integrating agents into production systems. The patterns apply across Claude, GPT, Gemini, and open models.
